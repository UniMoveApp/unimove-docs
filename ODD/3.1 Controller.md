# 3.1 Controller

Questo package contiene gli oggetti solution responsabili di esporre le API REST ai client frontend (Flutter). Intercettano le richieste HTTP e delegano la logica applicativa ai Service. Tutti i controller di questo layer sono layer di orchestrazione leggeri: estraggono l'identitĂ  dal token JWT tramite `Principal`, delegano la logica di business ai service e mappano le eccezioni dei service in risposte HTTP appropriate.

### Class: AuthController

**Mapping:** `@RestController @RequestMapping("/api/auth")`

**Descrizione:** Controller di autenticazione per UniMove che espone endpoint pubblici per login e registrazione. Tutti gli endpoint sotto `/api/auth/**` sono pubblicamente accessibili. L'autenticazione Ă¨ delegata a CINECA ESSE3, non a Spring Security.

**Dipendenze:** `AuthService` (iniettato via constructor injection).

**Operazioni:**

`login(request: LoginRequest): ResponseEntity<LoginResponse>`
- **Mapping:** `@PostMapping("/login")`
- **Descrizione:** Autentica un utente tramite CINECA ESSE3. Flusso: (1) validazione del payload, (2) delega dell'autenticazione ad `AuthService`, (3) recupero dei dati utente da CINECA, (4) generazione del token JWT.
- **Precondizioni:** `request.username` e `request.password` non devono essere vuoti.
- **Postcondizioni:** Restituisce `LoginResponse` con token JWT, username, ruolo e nome completo dell'utente.
- **Risposte HTTP:**
  - `200 OK`: login riuscito con corpo `LoginResponse`
  - `401 Unauthorized`: credenziali CINECA non valide o servizio CINECA irraggiungibile

`register(request: LoginRequest): ResponseEntity<LoginResponse>`
- **Mapping:** `@PostMapping("/register")`
- **Descrizione:** Registra un nuovo utente manuale con password gestita tramite BCrypt. Endpoint non utilizzato nel flusso principale di autenticazione tramite CINECA, mantenuto per scenari di test.
- **Precondizioni:** `request.username` non deve giĂ  esistere nel database.
- **Postcondizioni:** Restituisce `LoginResponse` con token JWT per il nuovo utente creato.
- **Risposte HTTP:**
  - `200 OK`: registrazione riuscita
  - `400 Bad Request`: username giĂ  esistente

`logout(): ResponseEntity<Void>`
- **Mapping:** `@PostMapping("/logout")`
- **Descrizione:** Endpoint placeholder per il logout. Non esegue alcuna operazione lato server poichĂ© il sistema Ă¨ completamente stateless â€” l'invalidazione della sessione avviene esclusivamente lato client tramite l'eliminazione del token JWT dal secure storage.
- **Precondizioni:** Nessuna.
- **Postcondizioni:** Nessuna modifica allo stato del server.
- **Risposte HTTP:**
  - `200 OK`: risposta vuota

### Class: UserController

**Mapping:** `@RestController @RequestMapping("/api/users")`

**Descrizione:** Controller responsabile dell'esposizione delle API relative al profilo utente. Tutti gli endpoint richiedono un token JWT valido. L'identitĂ  dell'utente viene sempre ricavata dal `Principal` popolato dal `JwtFilter`, mai da parametri nella request.

**Dipendenze:** `UserService` (iniettato via constructor injection).

**Operazioni:**

`getProfile(principal: Principal): ResponseEntity<UserProfileResponse>`
- **Mapping:** `@GetMapping("/me")`
- **Descrizione:** Restituisce il profilo completo dell'utente autenticato, incluse le corse imminenti come guidatore.
- **Precondizioni:** Il token JWT deve essere valido; l'utente deve esistere nel database.
- **Postcondizioni:** Restituisce `UserProfileResponse` con dati profilo e lista delle corse imminenti.
- **Risposte HTTP:**
  - `200 OK`: profilo recuperato con successo
  - `404 Not Found`: utente non trovato

`updatePreferences(principal: Principal, request: UpdatePreferenceRequest): ResponseEntity<Void>`
- **Mapping:** `@PutMapping("/me/preferences")`
- **Descrizione:** Aggiorna le preferenze di viaggio testuali dell'utente autenticato.
- **Precondizioni:** Il token JWT deve essere valido.
- **Postcondizioni:** Le preferenze di viaggio dell'utente sono aggiornate nel database.
- **Risposte HTTP:**
  - `200 OK`: aggiornamento riuscito

`updateIban(principal: Principal, request: UpdateIbanRequest): ResponseEntity<Void>`
- **Mapping:** `@PutMapping("/me/iban")`
- **Descrizione:** Aggiorna le informazioni bancarie dell'utente autenticato per la ricezione di donazioni volontarie.
- **Precondizioni:** Il token JWT deve essere valido.
- **Postcondizioni:** IBAN e intestatario del conto sono aggiornati nel database.
- **Risposte HTTP:**
  - `200 OK`: aggiornamento riuscito

`getRoutes(principal: Principal): ResponseEntity<List<RoutePreferenceResponse>>`
- **Mapping:** `@GetMapping("/me/routes")`
- **Descrizione:** Restituisce l'elenco delle tratte preferite dell'utente autenticato.
- **Precondizioni:** Il token JWT deve essere valido.
- **Postcondizioni:** Restituisce la lista delle tratte preferite (massimo 3).
- **Risposte HTTP:**
  - `200 OK`: elenco recuperato con successo

`addRoute(principal: Principal, request: RoutePreferenceRequest): ResponseEntity<RoutePreferenceResponse>`
- **Mapping:** `@PostMapping("/me/routes")`
- **Descrizione:** Aggiunge una nuova tratta preferita per l'utente autenticato.
- **Precondizioni:** L'utente non deve avere giĂ  3 tratte preferite registrate.
- **Postcondizioni:** La nuova tratta preferita Ă¨ salvata nel database e associata all'utente.
- **Eccezioni:** Lancia `InvalidRequestException("Massimo 3 tratte preferite consentite")` se il limite Ă¨ giĂ  raggiunto.
- **Risposte HTTP:**
  - `200 OK`: tratta aggiunta con successo
  - `400 Bad Request`: limite di tratte raggiunto

`deleteRoute(principal: Principal, id: UUID): ResponseEntity<Void>`
- **Mapping:** `@DeleteMapping("/me/routes/{id}")`
- **Descrizione:** Elimina una tratta preferita specifica dell'utente autenticato.
- **Precondizioni:** La tratta deve esistere ed appartenere all'utente autenticato.
- **Postcondizioni:** La tratta preferita Ă¨ eliminata dal database.
- **Eccezioni:** Lancia `ResourceNotFoundException("Tratta preferita non trovata")` se la tratta non esiste; lancia `UnauthorizedException("Non sei autorizzato a eliminare questa tratta preferita")` se la tratta appartiene a un altro utente.
- **Risposte HTTP:**
  - `200 OK`: eliminazione riuscita
  - `403 Forbidden`: tratta non appartenente all'utente
  - `404 Not Found`: tratta non trovata

### Class: RideController

**Mapping:** `@RestController @RequestMapping("/api/rides")`

**Descrizione:** Controller responsabile dell'esposizione delle API relative alle corse: creazione, ricerca, ciclo di vita (avvio, completamento, cancellazione) e archivio.

**Dipendenze:** `RideService` (iniettato via constructor injection).

**Operazioni:**

`createRide(principal: Principal, request: @Valid CreateRideRequest): ResponseEntity<RideResponse>`
- **Mapping:** `@PostMapping`
- **Descrizione:** Crea una nuova corsa associata all'utente autenticato come guidatore.
- **Precondizioni:** Il token JWT deve essere valido; `request` deve rispettare i vincoli di validazione (cittĂ  di partenza e arrivo obbligatorie, orario di partenza futuro, massimo 3 hotspot, numero di posti tra 1 e 8).
- **Postcondizioni:** La corsa Ă¨ salvata nel database con stato `OPEN` e posti disponibili pari al numero totale di posti.
- **Risposte HTTP:**
  - `200 OK`: corsa creata con successo, corpo `RideResponse`
  - `400 Bad Request`: payload non valido
  - `404 Not Found`: utente non trovato

`searchRides(driverUsername: String, departureCity: String, arrivalCity: String, date: LocalDate): ResponseEntity<List<RideResponse>>`
- **Mapping:** `@GetMapping("/search")` con `@RequestParam` opzionali
- **Descrizione:** Restituisce l'elenco delle corse disponibili filtrate secondo i criteri specificati. Tutti i parametri sono opzionali e combinabili liberamente.
- **Precondizioni:** Nessuna, tutti i parametri sono opzionali.
- **Postcondizioni:** Restituisce solo corse con stato `OPEN` e almeno un posto disponibile.
- **Risposte HTTP:**
  - `200 OK`: ricerca eseguita con successo, corpo lista di `RideResponse`

`startRide(principal: Principal, id: UUID): ResponseEntity<RideResponse>`
- **Mapping:** `@PutMapping("/{id}/start")`
- **Descrizione:** Avvia una corsa in stato `OPEN`, portandola in stato `IN_PROGRESS`. Solo il guidatore puĂ˛ eseguire questa operazione.
- **Precondizioni:** Il token JWT deve essere valido; la corsa deve esistere, essere in stato `OPEN` e appartenere all'utente autenticato.
- **Postcondizioni:** Lo stato della corsa Ă¨ aggiornato a `IN_PROGRESS`.
- **Risposte HTTP:**
  - `200 OK`: corsa avviata con successo
  - `400 Bad Request`: corsa non in stato `OPEN`
  - `403 Forbidden`: l'utente non Ă¨ il guidatore della corsa
  - `404 Not Found`: corsa non trovata

`completeRide(principal: Principal, id: UUID): ResponseEntity<RideResponse>`
- **Mapping:** `@PutMapping("/{id}/complete")`
- **Descrizione:** Completa una corsa in stato `IN_PROGRESS`, portandola in stato `COMPLETED`. Solo il guidatore puĂ˛ eseguire questa operazione.
- **Precondizioni:** Il token JWT deve essere valido; la corsa deve esistere, essere in stato `IN_PROGRESS` e appartenere all'utente autenticato.
- **Postcondizioni:** Lo stato della corsa Ă¨ aggiornato a `COMPLETED`.
- **Risposte HTTP:**
  - `200 OK`: corsa completata con successo
  - `400 Bad Request`: corsa non in stato `IN_PROGRESS`
  - `403 Forbidden`: l'utente non Ă¨ il guidatore della corsa
  - `404 Not Found`: corsa non trovata

`deleteRide(principal: Principal, id: UUID): ResponseEntity<Void>`
- **Mapping:** `@DeleteMapping("/{id}")`
- **Descrizione:** Elimina una corsa in stato `OPEN`. La cancellazione Ă¨ consentita solo con almeno 48 ore di anticipo rispetto alla partenza. Solo il guidatore puĂ˛ eseguire questa operazione.
- **Precondizioni:** Il token JWT deve essere valido; la corsa deve esistere, essere in stato `OPEN`, appartenere all'utente autenticato e la partenza deve essere a piĂą di 48 ore.
- **Postcondizioni:** La corsa Ă¨ eliminata dal database.
- **Risposte HTTP:**
  - `204 No Content`: eliminazione riuscita
  - `400 Bad Request`: corsa non in stato `OPEN` o partenza entro 48 ore
  - `403 Forbidden`: l'utente non Ă¨ il guidatore della corsa
  - `404 Not Found`: corsa non trovata

`getMyRides(principal: Principal, status: String): ResponseEntity<List<RideResponse>>`
- **Mapping:** `@GetMapping("/my")` con `@RequestParam` opzionale `status`
- **Descrizione:** Restituisce le corse del guidatore autenticato filtrate per stato. Se `status` non Ă¨ specificato, restituisce tutte le corse indipendentemente dallo stato.
- **Precondizioni:** Il token JWT deve essere valido.
- **Postcondizioni:** Restituisce la lista delle corse del guidatore ordinate per data di partenza decrescente.
- **Risposte HTTP:**
  - `200 OK`: lista recuperata con successo

`getArchive(principal: Principal): ResponseEntity<List<RideResponse>>`
- **Mapping:** `@GetMapping("/archive")`
- **Descrizione:** Restituisce l'archivio delle corse completate dell'utente autenticato, sia come guidatore che come passeggero, senza duplicati.
- **Precondizioni:** Il token JWT deve essere valido.
- **Postcondizioni:** Restituisce la lista delle corse completate ordinate per data di partenza decrescente.
- **Risposte HTTP:**
  - `200 OK`: archivio recuperato con successo

### Class: BookingController

**Mapping:** `@RestController @RequestMapping("/api/bookings")`

**Descrizione:** Controller responsabile dell'esposizione delle API relative alle prenotazioni delle corse. Tutti gli endpoint richiedono un token JWT valido. L'identitĂ  del passeggero viene sempre ricavata dal `Principal`.

**Dipendenze:** `BookingService` (iniettato via constructor injection).

**Operazioni:**

`createBooking(principal: Principal, request: @Valid CreateBookingRequest): ResponseEntity<BookingResponse>`
- **Mapping:** `@PostMapping`
- **Descrizione:** Crea una nuova prenotazione per la corsa specificata.
- **Precondizioni:** Il token JWT deve essere valido; `request.rideId` non deve essere null; la corsa deve essere in stato `OPEN`; il passeggero non deve essere il guidatore; non deve esistere giĂ  una prenotazione per la stessa corsa.
- **Postcondizioni:** La prenotazione Ă¨ salvata nel database e il campo `available_seats` della corsa Ă¨ decrementato atomicamente.
- **Risposte HTTP:**
  - `201 Created`: prenotazione creata con successo
  - `400 Bad Request`: corsa non disponibile, passeggero coincide con guidatore, prenotazione giĂ  esistente o nessun posto disponibile
  - `404 Not Found`: utente o corsa non trovata

`cancelBooking(principal: Principal, id: UUID): ResponseEntity<Void>`
- **Mapping:** `@DeleteMapping("/{id}")`
- **Descrizione:** Cancella una prenotazione esistente. Se la corsa Ă¨ in stato `OPEN`, la cancellazione richiede almeno 24 ore di anticipo. Se la corsa Ă¨ in stato `IN_PROGRESS`, l'abbandono Ă¨ sempre consentito.
- **Precondizioni:** Il token JWT deve essere valido; la prenotazione deve esistere e appartenere al passeggero autenticato; la corsa non deve essere in stato `COMPLETED`.
- **Postcondizioni:** La prenotazione Ă¨ eliminata e il campo `available_seats` della corsa Ă¨ incrementato atomicamente.
- **Risposte HTTP:**
  - `204 No Content`: cancellazione riuscita
  - `400 Bad Request`: cancellazione entro 24 ore dalla partenza o corsa giĂ  terminata
  - `404 Not Found`: prenotazione non trovata

`getMyBookings(principal: Principal): ResponseEntity<List<BookingResponse>>`
- **Mapping:** `@GetMapping("/my")`
- **Descrizione:** Restituisce tutte le prenotazioni del passeggero autenticato.
- **Precondizioni:** Il token JWT deve essere valido.
- **Postcondizioni:** Restituisce la lista delle prenotazioni del passeggero.
- **Risposte HTTP:**
  - `200 OK`: lista recuperata con successo

### Class: ReviewController

**Mapping:** `@RestController @RequestMapping("/api/reviews")`

**Descrizione:** Controller responsabile dell'esposizione delle API relative alle recensioni post-viaggio. Tutti gli endpoint richiedono un token JWT valido.

**Dipendenze:** `ReviewService` (iniettato via constructor injection).

**Operazioni:**

`createReview(principal: Principal, request: @Valid CreateReviewRequest): ResponseEntity<ReviewResponse>`
- **Mapping:** `@PostMapping`
- **Descrizione:** Crea una recensione per il guidatore di una corsa completata.
- **Precondizioni:** Il token JWT deve essere valido; la corsa deve essere in stato `COMPLETED`; il recensore deve essere stato passeggero della corsa; il recensore non deve aver giĂ  recensito la stessa corsa.
- **Postcondizioni:** La recensione Ă¨ salvata nel database.
- **Risposte HTTP:**
  - `201 Created`: recensione creata con successo
  - `400 Bad Request`: corsa non completata o recensione giĂ  esistente
  - `403 Forbidden`: l'utente non era passeggero della corsa
  - `404 Not Found`: utente o corsa non trovata

`getUserReviews(username: String): ResponseEntity<List<ReviewResponse>>`
- **Mapping:** `@GetMapping("/user/{username}")`
- **Descrizione:** Restituisce tutte le recensioni ricevute dall'utente identificato dallo username specificato.
- **Precondizioni:** Il token JWT deve essere valido; l'utente identificato da `username` deve esistere.
- **Postcondizioni:** Restituisce la lista delle recensioni ricevute dall'utente.
- **Risposte HTTP:**
  - `200 OK`: lista recuperata con successo
  - `404 Not Found`: utente non trovato
