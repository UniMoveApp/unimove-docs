# 3.4 Repository

Questo package contiene le interfacce per l'accesso ai dati persistenti tramite Spring Data JPA. Ogni interfaccia estende `JpaRepository`, delegando a Spring la generazione automatica delle query standard (CRUD di base).

## Interface: UserRepository

**Descrizione:** Repository per l'accesso ai dati dell'entità `User`.

**Estende:** `JpaRepository<User, UUID>`

**Operazioni:**

`findByUsername(username: String): Optional<User>`
- **Descrizione:** Cerca un utente tramite il suo username univoco.
- **Postcondizioni:** Restituisce un `Optional` contenente l'utente se trovato, altrimenti vuoto.

### Interface: RoutePreferenceRepository

**Descrizione:** Repository per l'accesso ai dati dell'entità `RoutePreference`.

**Estende:** `JpaRepository<RoutePreference, UUID>`

**Operazioni:**

`findByUser(user: User): List<RoutePreference>`
- **Descrizione:** Restituisce tutte le tratte preferite associate a un utente.

`countByUser(user: User): int`
- **Descrizione:** Conta il numero di tratte preferite associate a un utente, utilizzato per verificare il limite massimo di 3.

### Interface: RideRepository

**Descrizione:** Repository per l'accesso ai dati dell'entità `Ride`.

**Estende:** `JpaRepository<Ride, UUID>`

**Operazioni:**

`search(driverUsername: String, departureCity: String, arrivalCity: String, date: LocalDate): List<Ride>`
- **Descrizione:** Esegue una ricerca di corse tramite query JPQL personalizzata con filtri opzionali combinabili liberamente. Restituisce solo corse con stato `OPEN` e almeno un posto disponibile.
- **Precondizioni:** Tutti i parametri sono opzionali e possono essere `null`.

`findUpcomingByDriver(driver: User, now: LocalDateTime): List<Ride>`
- **Descrizione:** Restituisce le corse imminenti create da un guidatore specifico, con stato `OPEN` e orario di partenza successivo al momento corrente, ordinate per data crescente.
- **Precondizioni:** `driver` e `now` non devono essere null.

`findByDriverAndStatus(driver: User, status: String): List<Ride>`
- **Descrizione:** Restituisce le corse di un guidatore filtrate per stato tramite query JPQL personalizzata. Se `status` è `null`, restituisce tutte le corse indipendentemente dallo stato, ordinate per data di partenza decrescente.
- **Precondizioni:** `driver` non deve essere null; `status` è opzionale e può essere `null`.

`findCompletedByDriver(driver: User): List<Ride>`
- **Descrizione:** Restituisce le corse completate create da un guidatore specifico, ordinate per data di partenza decrescente.
- **Precondizioni:** `driver` non deve essere null.

`findCompletedByPassenger(passenger: User): List<Ride>`
- **Descrizione:** Restituisce le corse completate in cui l'utente era passeggero, tramite join con la tabella `bookings`, ordinate per data di partenza decrescente.
- **Precondizioni:** `passenger` non deve essere null.

`decrementAvailableSeats(rideId: UUID): int`
- **Descrizione:** Decrementa atomicamente di 1 il campo `available_seats` della corsa specificata, solo se il valore è maggiore di 0. Query `@Modifying` eseguita nella stessa transazione della creazione della prenotazione.
- **Postcondizioni:** Restituisce 1 se l'aggiornamento è avvenuto, 0 se non ci sono posti disponibili.

`incrementAvailableSeats(rideId: UUID): int`
- **Descrizione:** Incrementa atomicamente di 1 il campo `available_seats` della corsa specificata. Query `@Modifying` eseguita nella stessa transazione della cancellazione della prenotazione.
- **Postcondizioni:** Restituisce 1 se l'aggiornamento è avvenuto con successo.

### Interface: BookingRepository

**Descrizione:** Repository per l'accesso ai dati dell'entità `Booking`.

**Estende:** `JpaRepository<Booking, UUID>`

**Operazioni:**

`existsByRideAndPassenger(ride: Ride, passenger: User): boolean`
- **Descrizione:** Verifica se esiste già una prenotazione per la coppia corsa-passeggero specificata.
- **Postcondizioni:** Restituisce `true` se la prenotazione esiste, `false` altrimenti.

`findByPassenger(passenger: User): List<Booking>`
- **Descrizione:** Restituisce tutte le prenotazioni associate a un passeggero.
- **Precondizioni:** `passenger` non deve essere null.

`findByIdAndPassenger(id: UUID, passenger: User): Optional<Booking>`
- **Descrizione:** Cerca una prenotazione tramite il suo identificativo e il passeggero proprietario.
- **Postcondizioni:** Restituisce un `Optional` contenente la prenotazione se trovata e appartenente al passeggero, altrimenti vuoto.

---

### Interface: ReviewRepository

**Descrizione:** Repository per l'accesso ai dati dell'entità `Review`.

**Estende:** `JpaRepository<Review, UUID>`

**Operazioni:**

`existsByRideAndReviewer(ride: Ride, reviewer: User): boolean`
- **Descrizione:** Verifica se esiste già una recensione per la coppia corsa-recensore specificata.
- **Postcondizioni:** Restituisce `true` se la recensione esiste, `false` altrimenti.

`findByReviewed(reviewed: User): List<Review>`
- **Descrizione:** Restituisce tutte le recensioni ricevute da un utente specifico.
- **Precondizioni:** `reviewed` non deve essere null.
