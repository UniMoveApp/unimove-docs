# 3.5 Security

Questo package contiene il filtro JWT e le classi specifiche per la gestione dell'autenticazione stateless.

## Class: JwtUtil

**Descrizione:** Componente responsabile della generazione e validazione dei token JWT.

**Dipendenze:** Libreria jjwt 0.12.6 (`io.jsonwebtoken`).

**Attributi:**
- `secret: String` (configurato tramite `${jwt.secret}`) : chiave segreta HMAC per la firma dei token
- `expiration: long` (configurato tramite `${jwt.expiration}`) : durata di validità del token in millisecondi (864000000 ms = 10 giorni)

**Operazioni:**

`generateToken(username: String, role: String): String`
- **Descrizione:** Genera un token JWT firmato contenente username come subject e ruolo come claim personalizzato.
- **Precondizioni:** `username` e `role` non devono essere null.
- **Postcondizioni:** Restituisce un token JWT firmato con algoritmo HMAC-SHA384, valido per la durata configurata, contenente `subject` (username), claim `role`, `issuedAt` e `expiration`.

`extractUsername(token: String): String`
- **Descrizione:** Estrae lo username dal subject del token JWT.
- **Precondizioni:** `token` deve essere un JWT valido e correttamente firmato.
- **Postcondizioni:** Restituisce lo username contenuto nel subject del token.
- **Eccezioni:** Propaga le eccezioni di parsing della libreria jjwt se il token non è valido (gestite da `isTokenValid`).

`isTokenValid(token: String): boolean`
- **Descrizione:** Verifica se il token è valido : correttamente firmato e non scaduto.
- **Precondizioni:** Nessuna.
- **Postcondizioni:** Restituisce `true` se il token può essere validato con successo, `false` altrimenti (qualsiasi eccezione durante il parsing viene catturata e tradotta in `false`).

**Metodi privati di supporto:**

`getSigningKey(): SecretKey`
- **Descrizione:** Costruisce la chiave di firma HMAC a partire dal secret configurato, codificato in UTF-8.

### Class: JwtFilter

**Descrizione:** Filtro Spring che intercetta ogni richiesta HTTP in entrata per validare il token JWT e popolare il contesto di sicurezza. Estende `OncePerRequestFilter` per garantire l'esecuzione una sola volta per richiesta.

**Dipendenze:** `JwtUtil` (iniettato via constructor injection).

**Operazioni:**

`doFilterInternal(request: HttpServletRequest, response: HttpServletResponse, filterChain: FilterChain): void`
- **Descrizione:** Estrae il token dall'header `Authorization` (prefisso `Bearer `), lo valida tramite `JwtUtil` e, se valido, popola il `SecurityContext` di Spring con l'identità dell'utente.
- **Precondizioni:** Nessuna, il filtro gestisce anche il caso di richieste senza header `Authorization`, lasciandole passare senza autenticazione (saranno poi bloccate da Spring Security se l'endpoint richiede autenticazione).
- **Postcondizioni:** Se il token è valido, il `SecurityContext` viene popolato con un `UsernamePasswordAuthenticationToken` contenente lo username come principal e l'autorità fissa `ROLE_USER`.
- **Limitazione nota:** Il ruolo assegnato è sempre `ROLE_USER`, indipendentemente dal ruolo reale (`STUDENT`, `PROFESSOR`, `STAFF`) contenuto nel token. Questa limitazione è documentata anche in SDD 3.5 e sarà corretta in uno sprint successivo.
