# 3.4 Repository

Questo package contiene le interfacce per l'accesso ai dati persistenti tramite Spring Data JPA. Ogni interfaccia estende `JpaRepository`, delegando a Spring la generazione automatica delle query standard (CRUD di base).

## Interface: UserRepository

**Descrizione:** Repository per l'accesso ai dati dell'entità `User`.

**Estende:** `JpaRepository<User, UUID>`

**Operazioni:**

`findByUsername(username: String): Optional<User>`
- **Descrizione:** Cerca un utente tramite il suo username univoco.
- **Postcondizioni:** Restituisce un `Optional` contenente l'utente se trovato, altrimenti vuoto.

### Interface: RoutePreferenceRepository

**Descrizione:** Repository per l'accesso ai dati dell'entità `RoutePreference`.

**Estende:** `JpaRepository<RoutePreference, UUID>`

**Operazioni:**

`findByUser(user: User): List<RoutePreference>`
- **Descrizione:** Restituisce tutte le tratte preferite associate a un utente.

`countByUser(user: User): int`
- **Descrizione:** Conta il numero di tratte preferite associate a un utente, utilizzato per verificare il limite massimo di 3.

### Interface: RideRepository

**Descrizione:** Repository per l'accesso ai dati dell'entità `Ride`.

**Estende:** `JpaRepository<Ride, UUID>`

**Operazioni:**

`search(driverUsername: String, departureCity: String, arrivalCity: String, date: LocalDate): List<Ride>`
- **Descrizione:** Esegue una ricerca di corse tramite query JPQL personalizzata con filtri opzionali combinabili liberamente. Restituisce solo corse con stato `OPEN` e almeno un posto disponibile.
- **Precondizioni:** Tutti i parametri sono opzionali e possono essere `null`.

`findUpcomingByDriver(driver: User, now: LocalDateTime): List<Ride>`
- **Descrizione:** Restituisce le corse imminenti create da un guidatore specifico, con stato `OPEN` e orario di partenza successivo al momento corrente, ordinate per data crescente.
- **Precondizioni:** `driver` e `now` non devono essere null.
