# 3. Class Interfaces

Questa sezione descrive in dettaglio le classi solution e le loro interfacce pubbliche per il backend di UniMove. Per ogni classe sono specificate la responsabilità generale, le dipendenze, gli attributi e le operazioni pubbliche, incluse precondizioni, postcondizioni ed eccezioni secondo il principio Design by Contract.

## Panoramica dei Package

Questo capitolo è organizzato in 5 package principali:

- **3.1 Controller**: endpoint REST API (AuthController, UserController, RideController)
- **3.2 Service**: layer di logica di business (AuthService, UserService, RideService, RideMapper)
- **3.3 DTO**: Data Transfer Object per la comunicazione con il client
- **3.4 Repository**: interfacce di accesso ai dati persistenti
- **3.5 Security**: gestione e validazione dei token JWT

## Architettura Generale

Il backend di UniMove segue un'architettura a layer con netta separazione delle responsabilità:

- **Controller Layer**: Esposizione degli endpoint HTTP REST, mapping richiesta/risposta, delega ai service
- **Service Layer**: Implementazione della logica di business, orchestrazione delle operazioni di dominio, gestione del flusso di autenticazione
- **DTO Layer**: Definizione dei contratti dati per la comunicazione con il client, nasconde la complessità interna
- **Repository Layer**: Accesso ai dati persistenti tramite Spring Data JPA
- **Security Layer**: Generazione e validazione dei token JWT, filtro di autenticazione

Tutte le classi seguono il principio Design by Contract, con precondizioni, postcondizioni ed eccezioni esplicite per ogni operazione.
