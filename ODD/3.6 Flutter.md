# 3.6 Flutter Mobile Frontend

Questa sezione documenta le classi core del client mobile Flutter di UniMove. Seguono lo stesso principio Design by Contract adottato per il backend. Sono esclusi i widget puramente presentazionali (LoginScreen, HomeScreen, CreateRideScreen, MainScaffold) il cui comportamento è già descritto nei Sequence Diagram del RAD.

---

## AuthService

**Package:** `core/services/auth_service.dart`

**Responsabilità:** Gestisce la persistenza e la lettura del token JWT sul dispositivo tramite `flutter_secure_storage`. Fornisce inoltre la verifica dello stato di autenticazione e la gestione del flag di completamento dell'onboarding.

**Dipendenze:** `flutter_secure_storage`, `dart_jsonwebtoken`

**Attributi:**

| Attributo | Tipo | Visibilità | Descrizione |
|---|---|---|---|
| `_storage` | `FlutterSecureStorage` | `static const private` | Istanza dello storage sicuro del dispositivo |
| `_tokenKey` | `String` | `static const private` | Chiave per il token JWT nello storage |
| `_onboardingKey` | `String` | `static const private` | Chiave per il flag di onboarding nello storage |

**Operazioni:**

### `saveToken(token: String): Future<void>`
- **Precondizioni:** `token` non è null né vuoto
- **Postcondizioni:** Il token JWT è salvato in modo sicuro nello storage del dispositivo
- **Eccezioni:** Nessuna gestita esplicitamente — errori di storage propagati al chiamante

### `getToken(): Future<String?>`
- **Precondizioni:** Nessuna
- **Postcondizioni:** Restituisce il token JWT salvato, oppure `null` se assente
- **Eccezioni:** Nessuna

### `deleteToken(): Future<void>`
- **Precondizioni:** Nessuna
- **Postcondizioni:** Il token JWT e il flag di onboarding sono eliminati dallo storage. Usato durante il logout
- **Eccezioni:** Nessuna

### `setOnboardingCompleted(): Future<void>`
- **Precondizioni:** Nessuna
- **Postcondizioni:** Il flag di completamento onboarding è salvato nello storage con valore `'true'`
- **Eccezioni:** Nessuna

### `isOnboardingCompleted(): Future<bool>`
- **Precondizioni:** Nessuna
- **Postcondizioni:** Restituisce `true` se il flag di onboarding è presente e uguale a `'true'`, `false` altrimenti
- **Eccezioni:** Nessuna

### `isAuthenticated(): Future<bool>`
- **Precondizioni:** Nessuna
- **Postcondizioni:** Restituisce `true` se il token JWT è presente e non scaduto (verifica campo `exp` del payload). Restituisce `false` se il token è assente, scaduto o malformato
- **Eccezioni:** Qualsiasi eccezione di parsing del JWT viene catturata internamente — restituisce `false`

## ApiClient

**Package:** `core/services/api_client.dart`

**Responsabilità:** Configura e fornisce l'istanza `Dio` utilizzata per tutte le chiamate HTTP verso il backend. Tramite un `InterceptorsWrapper` aggiunge automaticamente il token JWT all'header `Authorization` di ogni richiesta e gestisce la risposta 401 eliminando il token scaduto.

**Dipendenze:** `Dio`, `AuthService`

**Attributi:**

| Attributo | Tipo | Visibilità | Descrizione |
|---|---|---|---|
| `_dio` | `Dio` | `private` | Istanza configurata del client HTTP |
| `_authService` | `AuthService` | `private` | Riferimento al servizio di autenticazione per la lettura del token |

**Operazioni:**

### `ApiClient(authService: AuthService)` *(costruttore)*
- **Precondizioni:** `authService` non è null
- **Postcondizioni:** L'istanza `Dio` è configurata con `baseUrl`, timeout e un `InterceptorsWrapper` che inietta il token JWT nell'header `Authorization: Bearer <token>` ad ogni richiesta e chiama `deleteToken()` in caso di risposta 401
- **Eccezioni:** Nessuna

### `dio: Dio` *(getter)*
- **Precondizioni:** Il costruttore è stato completato
- **Postcondizioni:** Restituisce l'istanza `Dio` configurata, pronta per effettuare chiamate HTTP
- **Eccezioni:** Nessuna

## AuthController

**Package:** `features/auth/presentation/auth_controller.dart`

**Responsabilità:** `StateNotifier` Riverpod che gestisce lo stato globale di autenticazione dell'applicazione. Coordina le operazioni di login, registrazione e logout invocando le API REST tramite `ApiClient` e aggiornando lo stato `AuthState` osservato dai widget.

**Dipendenze:** `ApiClient`, `AuthService`, `Riverpod StateNotifier`

**Attributi:**

| Attributo | Tipo | Visibilità | Descrizione |
|---|---|---|---|
| `_apiClient` | `ApiClient` | `private` | Client HTTP per le chiamate alle API di autenticazione |
| `_authService` | `AuthService` | `private` | Servizio per la persistenza del token JWT |

**Operazioni:**

### `_init(): Future<void>`
- **Precondizioni:** Nessuna — invocato automaticamente dal costruttore
- **Postcondizioni:** Lo stato è impostato a `AuthState.authenticated()` se il token locale è valido, altrimenti a `AuthState.unauthenticated()`
- **Eccezioni:** Nessuna

### `login(username: String, password: String): Future<bool>`
- **Precondizioni:** `username` e `password` non sono null né vuoti
- **Postcondizioni:** In caso di successo il token JWT è salvato tramite `AuthService` e lo stato è `AuthState.authenticated(isNewLogin: true)`. In caso di errore lo stato è `AuthState.unauthenticated(error: message)`
- **Eccezioni:** `DioException` catturata internamente — il messaggio di errore dalla risposta del backend viene propagato nello stato

### `register(username: String, password: String, fullName: String): Future<bool>`
- **Precondizioni:** `username`, `password` e `fullName` non sono null né vuoti
- **Postcondizioni:** In caso di successo il token JWT è salvato e lo stato è `AuthState.authenticated(isNewLogin: true)`. In caso di errore lo stato è `AuthState.unauthenticated(error: message)`
- **Eccezioni:** `DioException` catturata internamente

### `logout(): Future<void>`
- **Precondizioni:** Nessuna
- **Postcondizioni:** Il token JWT è eliminato tramite `AuthService` e lo stato è `AuthState.unauthenticated()`
- **Eccezioni:** Nessuna

### `completeWelcome(): Future<void>`
- **Precondizioni:** Lo stato corrente è `AuthState.authenticated`
- **Postcondizioni:** Il flag `isNewLogin` è impostato a `false` e il completamento dell'onboarding è persistito tramite `AuthService`
- **Eccezioni:** Nessuna

## UserProfile

**Package:** `features/auth/domain/user_profile.dart`

**Responsabilità:** Domain model che rappresenta il profilo completo dell'utente autenticato, deserializzato dalla risposta dell'endpoint `GET /api/users/me`.

**Attributi:**

| Attributo | Tipo | Visibilità | Descrizione |
|---|---|---|---|
| `username` | `String` | `public` | Username istituzionale dell'utente |
| `fullName` | `String` | `public` | Nome completo dell'utente |
| `email` | `String` | `public` | Email istituzionale |
| `role` | `String` | `public` | Ruolo: `STUDENT`, `PROFESSOR`, `STAFF` |
| `avatarUrl` | `String?` | `public` | URL avatar opzionale |
| `travelPreferences` | `String?` | `public` | Preferenze di viaggio opzionali |
| `iban` | `String?` | `public` | IBAN opzionale per ricevere donazioni |
| `ibanHolder` | `String?` | `public` | Intestatario del conto IBAN |
| `upcomingRides` | `List<Ride>` | `public` | Lista delle corse future come guidatore |

**Operazioni:**

### `UserProfile.fromJson(json: Map<String, dynamic>): UserProfile`
- **Precondizioni:** `json` non è null e contiene almeno i campi obbligatori (`username`, `fullName`, `email`, `role`)
- **Postcondizioni:** Restituisce un'istanza `UserProfile` con tutti i campi valorizzati. I campi assenti nel JSON vengono sostituiti con valori di default (`''` per le stringhe, `[]` per le liste, `null` per gli opzionali)
- **Eccezioni:** `TypeError` se i tipi nel JSON non corrispondono a quelli attesi

## Ride

**Package:** `features/auth/domain/user_profile.dart`

**Responsabilità:** Domain model che rappresenta una corsa, deserializzato dalla risposta del backend. Utilizzato sia nella lista `upcomingRides` del profilo utente che nelle risposte dell'endpoint di ricerca corse.

**Attributi:**

| Attributo | Tipo | Visibilità | Descrizione |
|---|---|---|---|
| `id` | `String` | `public` | UUID della corsa |
| `driverUsername` | `String` | `public` | Username del guidatore |
| `driverFullName` | `String` | `public` | Nome completo del guidatore |
| `departureCity` | `String` | `public` | Città di partenza |
| `departureTime` | `DateTime` | `public` | Data e ora di partenza |
| `arrivalCity` | `String` | `public` | Città di arrivo |
| `arrivalTimeEst` | `DateTime` | `public` | Orario stimato di arrivo |
| `hotspots` | `List<String>` | `public` | Lista di fermate intermedie opzionali (max 3) |
| `vehicleModel` | `String?` | `public` | Modello del veicolo opzionale |
| `vehiclePlate` | `String?` | `public` | Targa del veicolo opzionale |
| `totalSeats` | `int` | `public` | Numero totale di posti |
| `availableSeats` | `int` | `public` | Numero di posti ancora disponibili |
| `travelPreferences` | `String?` | `public` | Preferenze di viaggio del guidatore |
| `status` | `String` | `public` | Stato della corsa: `OPEN`, `FULL`, `COMPLETED`, `CANCELLED` |

**Operazioni:**

### `Ride.fromJson(json: Map<String, dynamic>): Ride`
- **Precondizioni:** `json` non è null e contiene almeno i campi obbligatori (`id`, `departureCity`, `arrivalCity`, `departureTime`, `arrivalTimeEst`, `totalSeats`, `availableSeats`, `status`)
- **Postcondizioni:** Restituisce un'istanza `Ride` completa. I campi assenti vengono sostituiti con valori di default. `departureTime` e `arrivalTimeEst` sono convertiti da `String` ISO 8601 a `DateTime`
- **Eccezioni:** `FormatException` se le stringhe di data non sono in formato ISO 8601 valido

## MyBookingsService (BookingController)

**Package:** `features/rides/presentation/my_bookings_controller.dart`

**Responsabilità:** Servizio che gestisce le azioni del client relative alle prenotazioni effettuate dai passeggeri, in particolare l'invocazione degli endpoint di cancellazione sul backend e l'invalidazione della cache dello stato.

**Dipendenze:** `ApiClient`, `Riverpod Provider`

**Attributi:**

| Attributo | Tipo | Visibilità | Descrizione |
|---|---|---|---|
| `_apiClient` | `ApiClient` | `private` | Client HTTP per le chiamate REST |
| `_ref` | `Ref` | `private` | Riferimento Riverpod per invalidare i provider a seguito delle azioni |

**Operazioni:**

### `MyBookingsService(apiClient: ApiClient, ref: Ref)` *(costruttore)*
- **Precondizioni:** `apiClient` e `ref` non sono null
- **Postcondizioni:** Inizializza il servizio con il client HTTP e il riferimento Riverpod
- **Eccezioni:** Nessuna

### `cancelBooking(bookingId: String): Future<void>`
- **Precondizioni:** `bookingId` non è null né vuoto
- **Postcondizioni:** Invia una richiesta `DELETE /bookings/<bookingId>` per cancellare la prenotazione. Se la chiamata ha successo, invalida `myBookingsProvider`, `userProfileProvider`, `myRidesProvider` e `archivedRidesProvider`
- **Eccezioni:** `Exception` se la risposta non è 200/204 o in caso di errore di rete `DioException` (estraendo l'errore specifico dal payload della risposta)

## ReviewsController (ReviewController)

**Package:** `features/profile/presentation/reviews_controller.dart`

**Responsabilità:** `StateNotifier` che gestisce lo stato di invio delle recensioni alle corse da parte dei passeggeri. Consente di inoltrare le valutazioni al backend e di tracciare se l'operazione asincrona è in corso.

**Dipendenze:** `ApiClient`, `Riverpod StateNotifier`

**Attributi:**

| Attributo | Tipo | Visibilità | Descrizione |
|---|---|---|---|
| `_apiClient` | `ApiClient` | `private` | Client HTTP per le chiamate REST |

**Operazioni:**

### `ReviewsController(apiClient: ApiClient)` *(costruttore)*
- **Precondizioni:** `apiClient` non è null
- **Postcondizioni:** Crea l'istanza impostando lo stato iniziale su `AsyncValue.data(null)`
- **Eccezioni:** Nessuna

### `leaveReview({required rideId: String, required rating: int, required comment: String}): Future<bool>`
- **Precondizioni:** `rideId` non è vuoto, `rating` è compreso tra 1 e 5
- **Postcondizioni:** Imposta lo stato a `AsyncValue.loading()`. Esegue una richiesta `POST /reviews` contenente l'ID della corsa, la valutazione e il commento. In caso di esito positivo ripristina lo stato a `AsyncValue.data(null)` e ritorna `true`. In caso di errore aggiorna lo stato a `AsyncError` e ritorna `false`
- **Eccezioni:** `DioException` catturata internamente e mappata nello stato di errore Riverpod

## PassengerBooking

**Package:** `features/rides/presentation/my_bookings_controller.dart`

**Responsabilità:** Domain model che rappresenta la prenotazione effettuata da un passeggero per una specifica corsa. Contiene i dettagli sull'hotspot prescelto e lo stato della prenotazione.

**Attributi:**

| Attributo | Tipo | Visibilità | Descrizione |
|---|---|---|---|
| `id` | `String` | `public` | UUID della prenotazione |
| `rideId` | `String` | `public` | UUID della corsa associata |
| `passengerUsername` | `String` | `public` | Username del passeggero |
| `passengerFullName` | `String` | `public` | Nome e cognome del passeggero |
| `hotspotChosen` | `String?` | `public` | Nome dell'hotspot di incontro scelto (opzionale) |
| `status` | `String` | `public` | Stato della prenotazione (es. `CONFIRMED`) |
| `createdAt` | `DateTime?` | `public` | Data e ora di creazione della prenotazione (opzionale) |
| `ride` | `Ride?` | `public` | Dettagli completi della corsa prenotata (associata in locale, opzionale) |

**Operazioni:**

### `PassengerBooking.fromJson(json: Map<String, dynamic>, {ride: Ride?}): PassengerBooking`
- **Precondizioni:** `json` non è null e contiene le chiavi obbligatorie (`id`, `rideId`, `passengerUsername`, `passengerFullName`)
- **Postcondizioni:** Restituisce un oggetto `PassengerBooking` popolato. I campi assenti vengono sostituiti con stringhe vuote o valori di default
- **Eccezioni:** `TypeError` in caso di formato dei dati non corrispondente

### `copyWith({ride: Ride?}): PassengerBooking`
- **Precondizioni:** Nessuna
- **Postcondizioni:** Restituisce una nuova istanza di `PassengerBooking` con i campi copiati dall'originale, sostituendo eventualmente l'oggetto `Ride`
- **Eccezioni:** Nessuna

## UserReview

**Package:** `features/auth/domain/user_profile.dart`

**Responsabilità:** Domain model che modella una recensione rilasciata sul profilo di un utente, contenente la valutazione numerica ed un testo descrittivo.

**Attributi:**

| Attributo | Tipo | Visibilità | Descrizione |
|---|---|---|---|
| `id` | `String?` | `public` | UUID identificativo della recensione (opzionale) |
| `rideId` | `String?` | `public` | UUID della corsa correlata (opzionale) |
| `reviewerUsername` | `String?` | `public` | Username dell'autore della recensione (opzionale) |
| `authorName` | `String` | `public` | Nome visualizzato dell'autore (risolto dal JSON) |
| `authorAvatar` | `String?` | `public` | URL dell'avatar dell'autore (opzionale) |
| `rating` | `double` | `public` | Valutazione espressa da 1.0 a 5.0 stelle |
| `comment` | `String` | `public` | Testo del commento rilasciato |
| `date` | `DateTime` | `public` | Data di creazione della recensione |

**Operazioni:**

### `UserReview.fromJson(json: Map<String, dynamic>): UserReview`
- **Precondizioni:** `json` non è null
- **Postcondizioni:** Restituisce un oggetto `UserReview` risolvendo l'autore tramite fallback (`reviewerFullName` -> `authorName` -> `reviewerUsername` -> `'Utente Anonimo'`) e popolando le date e il punteggio
- **Eccezioni:** `TypeError` o `FormatException` in caso di date non valide o tipi discordanti
