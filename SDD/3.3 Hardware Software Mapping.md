# 3.3 Hardware/Software Mapping

Questa sezione descrive come i sottosistemi identificati nel capitolo 3.2 vengono mappati sull'hardware e sulle tecnologie software concrete utilizzate nel progetto.

## Nodi Hardware

Il sistema UniMove è distribuito su tre nodi principali:

| Nodo | Descrizione |
|---|---|
| **Dispositivo mobile dell'utente** | Smartphone Android/IOS su cui viene eseguita l'applicazione Flutter. |
| **Server backend** | Macchina su cui viene eseguito il processo Spring Boot che espone le API REST. |
| **Server database** | Container Docker su cui viene eseguita l'istanza PostgreSQL 15. |

## Mapping Software

| Componente Software | Tecnologia | Nodo |
|---|---|---|
| Applicazione mobile | Flutter | Dispositivo mobile dell'utente |
| Backend API REST | Java 21 LTS, Spring Boot 4.0.6 | Server backend |
| Autenticazione e sicurezza | Spring Security, jjwt 0.12.6 | Server backend |
| Comunicazione con CINECA | RestClient (Spring Framework 6) | Server backend |
| Gestione migrazioni DB | Flyway | Server backend |
| Database relazionale | PostgreSQL 15 | Container Docker |
| Generazione codice boilerplate | Lombok | Server backend (compile time) |

## Ambiente di Sviluppo

Durante la fase di sviluppo, il database PostgreSQL viene eseguito in un container Docker tramite Docker Compose. Questo garantisce la consistenza dell'ambiente tra i membri del team, indipendentemente dal sistema operativo utilizzato. Il file `docker-compose.yml` presente nel repository consente di avviare l'intero ambiente con un singolo comando.

## Ambiente di Esecuzione

Il sistema viene eseguito interamente in locale durante la fase di sviluppo e testing. Il backend Spring Boot viene avviato direttamente tramite Maven o dall'IDE IntelliJ IDEA. Il database PostgreSQL viene avviato tramite Docker Compose con il comando `docker compose up -d`. La configurazione dell'ambiente è definita nel file `application.properties`, però in un ambiente di produzione reale, le credenziali andrebbero sostituite con variabili d'ambiente esterne al repository.

## Componenti Off-The-Shelf

Il sistema integra i seguenti componenti software già esistenti e collaudati:

| Componente | Ruolo nel sistema |
|---|---|
| Spring Boot 4.0.6 | Framework backend — gestisce API REST, sicurezza e persistenza |
| Flutter | Framework frontend — genera app mobile nativa Android/iOS |
| PostgreSQL 15 | DBMS relazionale per la persistenza dei dati |
| Docker | Containerizzazione del database in ambiente di sviluppo |
| Flyway | Gestione versionata delle migrazioni del database |
| jjwt 0.12.6 | Libreria per generazione e validazione token JWT |
| CINECA ESSE3 | Sistema esterno per l'autenticazione istituzionale |
