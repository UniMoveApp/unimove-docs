## 3.1 Overview

Il sistema UniMove adotta un'architettura **Client/Server** strutturata secondo il pattern **Three-tier** (a tre livelli logici), che garantisce una netta separazione delle responsabilità tra interfaccia utente, logica applicativa e persistenza dei dati:

- **Presentation Layer (Frontend):** Realizzato tramite un'applicazione mobile sviluppata in **Flutter**, responsabile in via esclusiva dell'interfaccia utente, della gestione reattiva dello stato e della comunicazione asincrona con i servizi remoti.
- **Application Logic Layer (Backend):** Realizzato in **Spring Boot** (Java 21 LTS), funge da punto di coordinamento centrale. Gestisce la logica di business, le regole di validazione e interagisce con sistemi esterni (CINECA). Il frontend comunica con il backend in modo esclusivo tramite **API REST in formato JSON** su protocollo HTTP/HTTPS.
- **Data Layer (Database):** Basato su **PostgreSQL 15**, utilizzato per la persistenza relazionale, sicura e strutturata dei dati applicativi. In linea con le best practice di sicurezza ed encapsulation, il backend non espone mai direttamente le entità del database al client, ma utilizza oggetti di trasferimento dati dedicati (**DTO - Data Transfer Objects**) sia per le richieste che per le risposte.
