# 4. Subsystem Services

Questa sezione descrive i servizi esposti dal backend di UniMove tramite API REST. Un servizio è definito come un insieme di operazioni correlate che condividono uno scopo comune. Le operazioni sono descritte a livello di interfaccia, comportamento, parametri richiesti e valori di ritorno attesi senza esporre i dettagli implementativi interni, documentati invece nell'ODD.

## 4.1 Servizio di Autenticazione

Questo servizio gestisce l'autenticazione degli utenti tramite le credenziali istituzionali CINECA ESSE3.

- `login(username: String, password: String): LoginResponse`
    - **Comportamento:** Valida le credenziali fornite tramite una chiamata proxy alle API ESSE3 di CINECA. Se la validazione ha successo, crea o aggiorna l'utente nel database locale e genera un token JWT firmato con HMAC-SHA384, valido 10 giorni.
    - **Parametri richiesti:** `username` (formato CINECA), `password`.
    - **Ritorno:** Restituisce un oggetto contenente il token JWT. In caso di credenziali non valide o servizio CINECA irraggiungibile, restituisce un errore HTTP appropriato senza esporre dettagli tecnici interni.

## 4.2 Servizio Profilo Utente

Questo servizio gestisce la visualizzazione e l'aggiornamento dei dati del profilo utente. Tutte le operazioni identificano l'utente a partire dal token JWT validato (`Principal`), non da un parametro nella request.

- `getProfile(username: String): UserProfileResponse`
  - **Comportamento:** Restituisce le informazioni del profilo dell'utente autenticato.
  - **Parametri richiesti:** Nessuno l'identità è ricavata dal token JWT.
  - **Ritorno:** Restituisce nome, cognome, email istituzionale, ruolo, preferenze di viaggio dell'utente autenticato.

- `updatePreferences(username: String, request: UpdatePreferenceRequest): void`
  - **Comportamento:** Aggiorna le preferenze di viaggio testuali dell'utente autenticato.
  - **Parametri richiesti:** Oggetto `UpdatePreferenceRequest`.
  - **Ritorno:** Nessun contenuto in caso di successo.

- `updateIban(username: String, request: UpdateIbanRequest): void`
  - **Comportamento:** Aggiorna le informazioni bancarie dell'utente per la ricezione di donazioni volontarie.
  - **Parametri richiesti:** Oggetto `UpdateIbanRequest` (IBAN e intestatario).
  - **Ritorno:** Nessun contenuto in caso di successo.

- `getRoutes(username: String): List<RoutePreferenceResponse>`
  - **Comportamento:** Restituisce l'elenco delle tratte preferite dell'utente autenticato.
  - **Parametri richiesti:** Nessuno.
  - **Ritorno:** Restituisce la lista delle tratte preferite (massimo 3).

- `addRoute(username: String, request: RoutePreferenceRequest): RoutePreferenceResponse`
  - **Comportamento:** Aggiunge una nuova tratta preferita per l'utente autenticato. Il sistema verifica che l'utente non abbia già raggiunto il limite massimo di 3 tratte preferite.
  - **Parametri richiesti:** Oggetto `RoutePreferenceRequest` (città di partenza e arrivo).
  - **Ritorno:** Restituisce la tratta preferita creata. Se l'utente ha già 3 tratte, restituisce un errore.

- `deleteRoute(username: String, id: UUID): void`
  - **Comportamento:** Elimina una tratta preferita specifica dell'utente autenticato.
  - **Parametri richiesti:** `id` della tratta da eliminare.
  - **Ritorno:** Nessun contenuto in caso di successo.

## 4.3 Servizio Corse

Questo servizio gestisce la creazione e la ricerca delle corse.

- `createRide(username: String, request: CreateRideRequest): RideResponse`
  - **Comportamento:** Crea una nuova corsa associata all'utente autenticato come guidatore. Lo stato iniziale della corsa è `OPEN` e i posti disponibili sono impostati pari al numero totale di posti specificato.
  - **Parametri richiesti:** Oggetto `CreateRideRequest` contenente città e orario di partenza, città e orario stimato di arrivo, fino a 3 hotspot intermedi opzionali, modello e targa del veicolo, numero totale di posti, preferenze di viaggio.
  - **Ritorno:** Restituisce la corsa creata, incluse le informazioni del guidatore.

- `searchRide(driverUsername: String, departureCity: String, arrivalCity: String, date: LocalDate): List<RideResponse>`
  - **Comportamento:** Restituisce l'elenco delle corse disponibili che soddisfano i criteri di ricerca specificati. Tutti i filtri sono opzionali e possono essere combinati liberamente. La ricerca restituisce solo corse con stato `OPEN` e almeno un posto disponibile.
  - **Parametri richiesti:** Tutti opzionali `driverUsername`, `departureCity`, `arrivalCity`, `date`.
  - **Ritorno:** Restituisce la lista delle corse corrispondenti ai criteri, incluse le informazioni del guidatore per ciascuna corsa.
