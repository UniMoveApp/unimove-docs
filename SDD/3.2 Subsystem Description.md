# 3.2 Subsystem Description

Il sistema UniMove è suddiviso in tre sottosistemi principali che comunicano tra loro secondo un'architettura a tre livelli.

## Frontend Flutter

Il sottosistema frontend è un'applicazione mobile sviluppata con il framework Flutter. Flutter è stato scelto per la sua capacità di generare applicazioni native per Android e iOS a partire da un'unica codebase, riducendo i tempi di sviluppo e garantendo consistenza dell'esperienza utente su entrambe le piattaforme. Il frontend comunica con il backend esclusivamente tramite API REST in formato JSON.

## Backend Spring Boot

Il sottosistema backend è sviluppato con il framework Spring Boot 4.0.6 su Java 21 LTS. Spring Boot è stato scelto per la sua maturità, per il supporto nativo alle API REST tramite Spring Web, per l'integrazione con Spring Security per la gestione dell'autenticazione e per Spring Data JPA per la comunicazione con il database. La libreria jjwt 0.12.6 è utilizzata per la generazione e validazione dei token JWT. La comunicazione con il sistema di autenticazione esterno CINECA ESSE3 avviene tramite RestClient di Spring Framework 6.

Il backend è organizzato secondo la seguente struttura a package:

| Package | Responsabilità |
|---|---|
| `controller` | Riceve le richieste HTTP dal client Flutter e le instrada al service corretto. Non contiene logica di business. |
| `service` | Contiene tutta la logica di business. Il controller chiama il service, non è il contrario. |
| `repository` | Interfacce Spring Data JPA per la comunicazione con il database. |
| `model` | Entità JPA che mappano le tabelle del database. |
| `dto` | Data Transfer Object sono gli oggetti usati per ricevere dati in entrata e restituire dati in uscita dalle API. Le entità del model non vengono mai esposte direttamente. |
| `config` | Configurazioni Spring: Security, CORS e altro. |
| `security` | Filtro JWT e classi specifiche per l'autenticazione. |
| `exception` | Gestione centralizzata degli errori tramite GlobalExceptionHandler. |

## Database — PostgreSQL

Il sottosistema database è PostgreSQL 15. È stato scelto un database relazionale per garantire l'integrità dei dati tramite vincoli di chiave esterna e per il supporto nativo al tipo UUID, utilizzato come chiave primaria in tutte le tabelle. La gestione dello schema è delegata interamente a Flyway.

## Interfacce tra Sottosistemi

| Interfaccia | Descrizione |
|---|---|
| Flutter → Spring Boot | Il client mobile comunica con il backend tramite API REST in formato JSON su protocollo HTTP/HTTPS. |
| Spring Boot → PostgreSQL | Il backend comunica con il database tramite Spring Data JPA e il driver JDBC PostgreSQL. |
| Spring Boot → CINECA ESSE3 | Il backend comunica con il sistema di autenticazione esterno tramite RestClient con Basic Auth. |


 **Nota:** Il progetto utilizza la libreria **Lombok** per la generazione automatica di getter, setter, costruttori e builder, riducendo il codice boilerplate nelle classi model e dto.
