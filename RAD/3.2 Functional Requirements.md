# 3.2 Functional Requirements

I requisiti funzionali di **UniMove** sono organizzati in moduli logici che riflettono le principali aree operative dell'applicazione.

---

### Modulo 1: Autenticazione e Accesso

| ID     | Requisito                     | Descrizione                                                                                                                                       |
| ------ | ----------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------- |
| RF-1.1 | Login istituzionale | Il sistema deve consentire l'accesso esclusivamente tramite credenziali istituzionali UNIMOL fornite dal sistema CINECA. |
| RF-1.2 | Accettazione Condizioni d'Uso | Il sistema deve richiedere l'accettazione esplicita delle Condizioni d'Uso e della Privacy Policy prima di consentire l'accesso all'applicazione. |
| RF-1.3 | Logout                        | Il sistema deve consentire all'utente di effettuare il logout, previa conferma tramite finestra di dialogo, invalidando la sessione lato client.  |

### Modulo 2: Onboarding

| ID     | Requisito                  | Descrizione                                                                                                                                                    |
| ------ | -------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| RF-2.1 | Selezione tratte preferite | Il sistema deve consentire all'utente, al primo accesso, di selezionare fino a un massimo di 3 tratte preferite per ricevere notifiche sulle corse disponibili. |
| RF-2.2 | Skip onboarding            | Il sistema deve consentire all'utente di saltare la fase di configurazione delle tratte preferite, rimandando la scelta a un momento successivo.               |
### Modulo 3: Gestione Profilo Utente

| ID     | Requisito                      | Descrizione                                                                                                                                                         |
| ------ | ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| RF-3.1 | Visualizzazione profilo | Il sistema deve mostrare sul profilo dell'utente le informazioni istituzionali: nome, cognome, email istituzionale e ruolo (studente, docente, dipendente). |
| RF-3.2 | Preferenze di viaggio          | Il sistema deve consentire all'utente di impostare e modificare le proprie preferenze di viaggio, visibili agli altri utenti sul proprio profilo.                  |
| RF-3.3 | Gestione tratte preferite      | Il sistema deve consentire all'utente di aggiungere, modificare ed eliminare le proprie tratte preferite, fino a un massimo di 3.                                  |
| RF-3.4 | Gestione informazioni bancarie | Il sistema deve consentire all'utente di inserire e aggiornare il proprio IBAN e l'intestatario del conto per ricevere donazioni volontarie.                       |
| RF-3.5 | Visualizzazione recensioni     | Il sistema deve mostrare sul profilo di ciascun utente le recensioni ricevute dagli altri utenti, con relativa valutazione a stelle.                               |
| RF-3.6 | Segnalazione utente            | Il sistema deve consentire all'utente di segnalare un altro utente selezionando una motivazione tra quelle disponibili.                                            |

### Modulo 4: Gestione Corse (Guidatore)

| ID     | Requisito                        | Descrizione                                                                                                                                                                      |
| ------ | -------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| RF-4.1 | Creazione corsa                  | Il sistema deve consentire al guidatore di creare una corsa inserendo data, città di partenza, orario di partenza, città di arrivo, orario stimato di arrivo e numero di posti. |
| RF-4.2 | Inserimento hotspot              | Il sistema deve consentire al guidatore di specificare fino a 3 punti di incontro intermedi (hotspot) opzionali lungo il tragitto.                                               |
| RF-4.3 | Inserimento dati veicolo         | Il sistema deve richiedere al guidatore l'inserimento del modello e della targa del veicolo utilizzato per la corsa.                                                             |
| RF-4.4 | Preferenze di viaggio della corsa| Il sistema deve consentire al guidatore di impostare le proprie preferenze di viaggio per la corsa (es. musica, fumo).                                                          |
| RF-4.5 | Inserimento IBAN al primo evento | Il sistema deve richiedere al guidatore, alla creazione del primo evento, di inserire facoltativamente le informazioni bancarie per ricevere eventuali donazioni.                |
| RF-4.6 | Avvio corsa                      | Il sistema deve consentire al guidatore di avviare una corsa programmata tramite apposito comando.                                                                               |
| RF-4.7 | Eliminazione corsa | Il sistema deve consentire al guidatore di annullare una corsa programmata con un preavviso minimo di 48 ore rispetto all'orario di partenza previsto. |
| RF-4.8 | Tracciamento passeggeri | Il sistema deve consentire al guidatore, una volta avviata la corsa, di visualizzare la posizione condivisa dai passeggeri accettati per facilitarne il raggiungimento. |

### Modulo 5: Ricerca e Prenotazione Corse (Passeggero)

| ID     | Requisito                         | Descrizione                                                                                                                                                                         |
| ------ | --------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| RF-5.1 | Ricerca corsa                     | Il sistema deve consentire al passeggero di cercare una corsa filtrando per username guidatore, data, città di partenza, orario di partenza, città di arrivo, numero di posti disponibili  e preferenze.  |
| RF-5.2 | Visualizzazione risultati         | Il sistema deve mostrare l'elenco delle corse disponibili corrispondenti ai criteri di ricerca, con i dettagli del guidatore e del tragitto.                                        |
| RF-5.3 | Selezione hotspot | Il sistema deve consentire al passeggero di selezionare il proprio punto di incontro tra gli hotspot definiti dal guidatore prima di inviare la richiesta di prenotazione. |
| RF-5.4 | Prenotazione corsa | Il sistema deve consentire al passeggero di inviare una richiesta di prenotazione a una corsa, includendo l'hotspot di incontro selezionato. I posti vengono occupati in ordine di richiesta solo dopo l'approvazione del guidatore, fino ad esaurimento della disponibilità. |
| RF-5.5 | Lista d'attesa                    | Il sistema deve consentire al passeggero di iscriversi alla lista d'attesa di una corsa al completo, notificandolo automaticamente in caso di posto libero.                        |
| RF-5.6 | Cancellazione prenotazione | Il sistema deve consentire al passeggero di cancellarsi da una corsa con un preavviso minimo di 24 ore rispetto all'orario di partenza previsto. |
| RF-5.7 | Tracciamento corsa                | Il sistema deve consentire al passeggero di visualizzare la posizione in tempo reale del guidatore durante la corsa tramite integrazione con servizi di mappe.                     |
| RF-5.8 | Abbandono corsa                   | Il sistema deve consentire al passeggero di abbandonare una corsa a cui è prenotato.                                                                                               |

### Modulo 6: Home e Gestione Eventi

| ID     | Requisito                    | Descrizione                                                                                                                                                 |
| ------ | ---------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| RF-6.1 | Promemoria eventi            | Il sistema deve mostrare nella schermata principale un promemoria con gli eventi in programma nelle prossime ore o nel giorno successivo.                   |
| RF-6.2 | I miei eventi                | Il sistema deve mostrare al guidatore l'elenco delle corse da lui create, con la possibilità di avviarle o eliminarle.                                      |
| RF-6.3 | Prenotazioni                 | Il sistema deve mostrare al passeggero l'elenco delle corse a cui è prenotato, con la possibilità di tracciarle o abbandonarle.                             |
| RF-6.4 | Archivio                     | Il sistema deve mostrare l'elenco delle corse terminate, consentendo al passeggero di effettuare una donazione o lasciare una recensione al guidatore.      |

### Modulo 7: Chat

| ID     | Requisito                    | Descrizione                                                                                                                                                          |
| ------ | ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| RF-7.1 | Chat privata                 | Il sistema deve creare automaticamente una chat privata tra guidatore e passeggero nel momento in cui il passeggero viene accettato per una corsa.                   |
| RF-7.2 | Condivisione posizione | Il sistema deve consentire al passeggero di inviare la propria posizione GPS in chat al guidatore per concordare il punto di incontro. |
| RF-7.3 | Eliminazione chat            | Il sistema deve eliminare automaticamente la chat tra guidatore e passeggero al termine della corsa.                                                                 |

### Modulo 8: Notifiche

| ID     | Requisito                        | Descrizione                                                                                                                                                     |
| ------ | -------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| RF-8.1 | Notifica passaggio accettato     | Il sistema deve notificare il passeggero quando il guidatore accetta la sua richiesta di partecipazione a una corsa.                                            |
| RF-8.2 | Notifica annullamento evento     | Il sistema deve notificare tutti i passeggeri prenotati quando il guidatore annulla una corsa.                                                                  |
| RF-8.3 | Notifica nuova recensione        | Il sistema deve notificare l'utente quando riceve una nuova recensione da parte di un altro utente.                                                             |
| RF-8.4 | Notifica posto liberato          | Il sistema deve notificare il primo utente in lista d'attesa quando si libera un posto in una corsa al completo.                                                |
| RF-8.5 | Notifica nuova corsa disponibile | Il sistema deve notificare l'utente quando viene pubblicata una nuova corsa su una delle sue tratte preferite.                                                  |
| RF-8.6 | Notifica richiesta di passaggio  | Il sistema deve notificare il guidatore quando un passeggero richiede di partecipare alla sua corsa, indicando l'hotspot scelto, con possibilità di accettare o rifiutare. |
| RF-8.7 | Notifica di benvenuto | Il sistema deve inviare una notifica di benvenuto all'utente al momento del primo accesso all'applicazione. |

### Modulo 9: Recensioni e Donazioni

| ID     | Requisito                    | Descrizione                                                                                                                                                          |
| ------ | ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| RF-9.1 | Recensione guidatore         | Il sistema deve consentire al passeggero di lasciare una recensione con valutazione a stelle al guidatore al termine della corsa, una sola volta per corsa.          |
| RF-9.2 | Donazione volontaria | Il sistema deve consentire al passeggero di effettuare una donazione volontaria al guidatore al termine della corsa, inserendo i dati della propria carta di pagamento (incluso CVC) per inviare il contributo sul conto IBAN associato al guidatore. |
